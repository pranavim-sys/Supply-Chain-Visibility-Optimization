Supply Chain Visibility and Optimization

Objective:
Create a Fact Table for core transactional data.
Build Dimension Tables (Customer, Product, Category, Shipping, Location, Department, Date).
Establish relationships between fact and dimension tables for better modeling.
Perform data cleaning and transformation (removing duplicates, sensitive columns, renaming IDs).
Enable visibility into order and shipping timelines.
Optimize reporting for supply chain performance.

Dataset Source:
Source: Kaggle
The dataset contains information related to supply chain operations such as orders, shipping, locations, and customer details.
It is used to build fact and dimension tables, enabling structured analysis and reporting.
